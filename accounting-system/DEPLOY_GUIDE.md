# คู่มือการติดตั้งและขึ้นระบบบน Google Cloud Run (Deployment Guide)

คู่มือนี้แนะนำขั้นตอนการนำระบบบัญชีของคุณขึ้นไปรันออนไลน์บนคลาวด์ด้วยบริการ **Google Cloud Run** ซึ่งรวดเร็ว ปลอดภัย และมีค่าใช้จ่ายเริ่มต้นที่ถูกมาก (มีโควต้าให้ใช้ฟรีทุกเดือน)

---

## สิ่งที่ต้องเตรียมก่อนเริ่ม (Prerequisites)
1. **Google Cloud Account**: สมัครใช้งานบัญชี Google Cloud Platform (GCP)
2. **Google Cloud SDK**: ติดตั้งเครื่องมือควบคุมผ่านคอมพิวเตอร์ของคุณ (gcloud CLI) โดยสามารถดาวน์โหลดได้จาก: [https://cloud.google.com/sdk/docs/install](https://cloud.google.com/sdk/docs/install)

---

## ขั้นตอนการขึ้นระบบ (Deployment Steps)

### ขั้นตอนที่ 1: เข้าสู่ระบบและตั้งโปรเจกต์ผ่าน Terminal
เปิดโปรแกรม **Terminal** ใน Mac ของคุณ แล้วรันคำสั่งเหล่านี้ทีละแถว:

1. ล็อกอินเข้าสู่ระบบบัญชี Google Cloud:
   ```bash
   gcloud auth login
   ```
2. ย้ายตำแหน่ง Terminal ไปยังโฟลเดอร์ของโปรเจกต์:
   ```bash
   cd ~/Documents/accounting-system
   ```
3. กำหนดเป้าหมายการทำงานไปยังโปรเจกต์ GCP ของคุณ (แทนที่ `[PROJECT_ID]` ด้วยไอดีโปรเจกต์ของคุณ):
   ```bash
   gcloud config set project [PROJECT_ID]
   ```

---

### ขั้นตอนที่ 2: Build คอนเทนเนอร์และอัปโหลดขึ้น Cloud (Google Cloud Builds)
รันคำสั่งด้านล่างนี้เพื่อส่งไฟล์โค้ดไปประกอบขึ้นเป็น Container Image บนระบบคลาวด์ของ Google:

```bash
gcloud builds submit --tag gcr.io/[PROJECT_ID]/accounting-system
```
*(ระบบจะทำหน้าที่อัปโหลดไฟล์ และทำการ Build ไฟล์ตามชุดคำสั่งใน `Dockerfile` ที่ผมสร้างไว้ให้โดยอัตโนมัติ)*

---

### ขั้นตอนที่ 3: สั่งให้ระบบทำงานบน Cloud Run (Deploy)
รันคำสั่งสุดท้ายด้านล่างนี้เพื่อสร้างเซิร์ฟเวอร์และให้บริการออนไลน์:

```bash
gcloud run deploy accounting-system \
  --image gcr.io/[PROJECT_ID]/accounting-system \
  --platform managed \
  --allow-unauthenticated \
  --region asia-southeast1
```

**ตัวเลือกคำสั่งเพิ่มเติมที่คุณต้องเลือกตอบระหว่างรัน:**
- หากระบบถามว่าต้องการเปิดใช้งาน API ที่จำเป็น เช่น `artifactregistry.googleapis.com` หรือ `run.googleapis.com` หรือไม่? ให้ตอบ **`y` (Yes)**
- เมื่อ Deploy เสร็จสิ้น ระบบจะพิมพ์ข้อความแสดงสถานะความสำเร็จและให้ลิงก์สีเขียวมา เช่น:
  `Service [accounting-system] revision [accounting-system-00001-xxx] has been deployed and is serving 100% of traffic.`
  `Service URL: https://accounting-system-xxxxxx-as.a.run.app`

---

## คำแนะนำเกี่ยวกับการทำงานของฐานข้อมูล
เนื่องจากระบบปัจจุบันทำงานบนฐานข้อมูล **IndexedDB** เมื่อคุณแชร์ลิงก์ให้ทีมงานใช้งาน:
1. แต่ละคนจะเปิดใช้งานระบบได้ปกติผ่านบราวเซอร์ของตนเอง
2. **ข้อมูลจะไม่แชร์ร่วมกันข้ามเครื่อง:** ข้อมูลที่ลูกค้าคีย์จะบันทึกอยู่ในเครื่องของลูกค้า ข้อมูลที่คุณคีย์จะบันทึกอยู่ในบราวเซอร์เครื่องของคุณ
3. **การนำข้อมูลมารวมกัน:** หากต้องการรวมบัญชี สามารถใช้ปุ่ม **ดาวน์โหลดสำรองข้อมูล (Export JSON)** ในเมนูตั้งค่า แล้วส่งให้อีกฝ่ายกด **นำเข้าข้อมูล (Import JSON)** เพื่อเปิดใช้งานร่วมกันได้ครับ

---

# ภาคพิเศษ: การติดตั้งใช้งานบนเว็บฟรี (Free Web Hosting Guide)

เนื่องจากระบบบัญชีนี้ถูกออกแบบให้เป็น **Local-first (ทำงานฝั่งผู้ใช้ทั้งหมด)** จึงมีเพียงไฟล์ static (HTML, CSS, JS) และไม่มีเซิร์ฟเวอร์ฐานข้อมูลหลังบ้าน ทำให้เราสามารถเอาขึ้นเว็บเพื่อให้ใช้งานออนไลน์ได้ **ฟรี 100% ตลอดชีพ** โดยใช้บริการดังต่อไปนี้ครับ:

## วิธีที่ 1: ติดตั้งผ่าน Netlify Drop (ง่ายที่สุดในโลก - ไม่ต้องเขียนโค้ด/ไม่ต้องสมัครสมาชิกก็ทำได้)

1. เปิดเว็บเบราว์เซอร์ไปที่ลิงก์: 👉 **[https://app.netlify.com/drop](https://app.netlify.com/drop)**
2. เปิดหน้าต่าง File Explorer ในคอมพิวเตอร์ของคุณ แล้วเข้าไปที่โฟลเดอร์โครงการ `accounting-system`
3. **ลากและวาง (Drag & Drop)** โฟลเดอร์ `accounting-system` ทั้งโฟลเดอร์ลงในกล่องอัปโหลดบนหน้าเว็บ Netlify
4. รอระบบประมวลผลประมาณ 3-5 วินาที คุณจะได้ลิงก์สำหรับเข้าใช้งานระบบทันที (เช่น `https://cool-cookie-12345.netlify.app`)
5. *ข้อแนะนำ:* คุณสามารถสมัครสมาชิก Netlify (ฟรี) เพื่อเข้าไปกดเปลี่ยนชื่อลิงก์เว็บให้สวยงามตามชื่อบริษัทของคุณได้ เช่น `https://my-company-accounting.netlify.app`

---

## วิธีที่ 2: ติดตั้งผ่าน Vercel (เร็ว เสถียร และเป็นที่นิยมสูง)

1. สมัครใช้งานบัญชีฟรีที่เว็บ: 👉 **[https://vercel.com](https://vercel.com)**
2. เมื่อเข้าสู่หน้าแดชบอร์ดหลักของ Vercel ให้ไปที่ส่วน **Vercel Ship** หรือตัวเลือกอัปโหลดโปรเจกต์แบบลากวาง
3. ทำการลากโฟลเดอร์ `accounting-system` ทั้งโฟลเดอร์ไปวางเพื่ออัปโหลด
4. Vercel จะสร้างโปรเจกต์และปล่อยลิงก์เข้าใช้งานแบบฟรีมาให้คุณทันที (เช่น `https://accounting-system.vercel.app`)

---

## วิธีที่ 3: ติดตั้งผ่าน Cloudflare Pages (แนะนำ - ฟรี 100% รวดเร็ว และเสถียรที่สุด)

เนื่องจากระบบต้องการเพียงการฝากไฟล์หน้าเว็บ (Static Assets) การใช้ **Cloudflare Pages** จึงเป็นทางเลือกที่ดีที่สุดในการเปิดใช้งานหน้าเว็บออนไลน์ เพราะเปิดให้บริการฟรี ไม่จำกัดแบนด์วิดท์ และรันด้วยเน็ตเวิร์กที่เร็วที่สุดในโลก โดยทำได้ง่ายๆ ผ่านหน้าเว็บเบราว์เซอร์ดังนี้:

1. **เตรียมไฟล์สำหรับอัปโหลด:**
   * บีบอัดโฟลเดอร์โครงการเป็นไฟล์ ZIP (นำไฟล์ `index.html`, โฟลเดอร์ `css`, โฟลเดอร์ `js` เข้าไปอยู่ในไฟล์ ZIP โดยตรง)
2. **เข้าสู่ระบบ Cloudflare:**
   * สมัครสมาชิกหรือล็อกอินเข้าใช้งาน: 👉 **[https://dash.cloudflare.com](https://dash.cloudflare.com)**
3. **เริ่มโครงการ Pages:**
   * ที่แถบเมนูด้านซ้าย เลือกเมนู **"Workers & Pages"**
   * คลิกปุ่ม **"Create Application"**
   * เลือกแท็บย่อย **"Pages"** ด้านบน แล้วคลิกปุ่ม **"Upload assets"** (Direct Upload)
4. **ตั้งชื่อและอัปโหลด:**
   * กรอกชื่อโครงการที่คุณต้องการ (เช่น `accounting-system`) -> คลิก **"Create project"**
   * ทำการลากไฟล์ **ZIP** ที่เตรียมไว้ในขั้นตอนที่ 1 ไปวางลงในกล่องอัปโหลด
   * เมื่ออัปโหลดเสร็จสมบูรณ์ ให้คลิกปุ่ม **"Deploy site"**
5. **เข้าใช้งานระบบ:**
   * ระบบจะสร้างลิงก์สำหรับเข้าใช้งานให้ฟรีทันที เช่น `https://[ชื่อโปรเจกต์].pages.dev` (เช่น `https://accounting-system.pages.dev`)

---

## 🔒 ข้อควรทราบในเรื่องความปลอดภัยของข้อมูลเมื่อนำขึ้นเว็บ
* **ข้อมูลของคุณยังคงปลอดภัย 100%:** แม้ว่าหน้าเว็บจะรันอยู่บนอินเทอร์เน็ต แต่ข้อมูลทางบัญชี รายการเดินบัญชี ลูกหนี้ และเจ้าหนี้ทั้งหมดจะถูกบันทึกไว้ใน **IndexedDB ภายในเครื่องคอมพิวเตอร์และเบราว์เซอร์ของผู้ใช้งานแต่ละคนเท่านั้น** 
* **ไม่มีการรั่วไหลขึ้นเซิร์ฟเวอร์:** ผู้ให้บริการโฮสติ้ง (Netlify/Vercel) จะทำหน้าที่เพียงส่งไฟล์หน้าเว็บเปล่าๆ มาให้เครื่องคุณรัน ดังนั้น สรรพากร ผู้ให้บริการ หรือคนนอก จะไม่มีสิทธิ์มองเห็นข้อมูลบัญชีและการเงินของคุณได้เลยครับ
